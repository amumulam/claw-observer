## 1. 项目概述

### 1.1 背景

需要实时监控 OpenClaw Gateway 的运行状态，但受限于以下核心约束：

| 约束 | 说明 |
|------|------|
| **零代码侵入** | 不能修改 OpenClaw 源码，需追踪上游版本更新 |
| **高实时性** | 状态变化需毫秒级推送到本地终端 |
| **低开销** | 避免频繁磁盘 I/O，全内存流转 |

### 1.2 解决方案

采用 **Sidecar（边车模式）+ 内存状态机 + WebSocket** 架构：

```
┌─────────────────────────────────────────────────────────────┐
│                      远端服务器                              │
│  ┌─────────────────┐         ┌───────────────────────────┐  │
│  │                 │ 日志流   │                           │  │
│  │ OpenClaw Gateway├────────►│ Sidecar Observer Service   │  │
│  │                 │ (stdout)│  ┌─────────────────────┐  │  │
│  │                 │         │  │ 日志流读取器         │  │  │
│  │                 │         │  └──────────┬──────────┘  │  │
│  │                 │         │  ┌──────────▼──────────┐  │  │
│  │                 │         │  │ 正则解析器 + 规则引擎 │  │  │
│  │                 │         │  └──────────┬──────────┘  │  │
│  │                 │         │  ┌──────────▼──────────┐  │  │
│  │                 │         │  │ 内存状态机           │  │  │
│  │                 │         │  │ IDLE→THINKING→      │  │  │
│  │                 │         │  │ EXECUTING→REPLYING  │  │  │
│  │                 │         │  └──────────┬──────────┘  │  │
│  │                 │         │  ┌──────────▼──────────┐  │  │
│  │                 │         │  │ WebSocket Server    │  │  │
│  │                 │         │  │ Port: 8765          │  │  │
│  │                 │         │  └──────────┬──────────┘  │  │
│  └─────────────────┼─────────┼─────────────┼─────────────┘  │
└────────────────────┼─────────┼─────────────┼─────────────────┘
                     │         │             │
                     │ SSH 隧道 / TLS         │
                     ▼         ▼             ▼
┌─────────────────────────────────────────────────────────────┐
│                      本地机器                                │
│  ┌─────────────────────────────────────────────────────────┐│
│  │  CLI Terminal UI (Rich / Textual)                       ││
│  │  ┌─────────────────────────────────────────────────────┐││
│  │  │  WebSocket Client → 状态渲染 → 实时刷新              │││
│  │  └─────────────────────────────────────────────────────┘││
│  └─────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────┘
```

### 1.3 核心目标

| 目标 | 验收标准 |
|------|----------|
| 实时捕获日志流 | 延迟 < 100ms |
| 准确解析状态 | 5 种状态正确率 > 99% |
| 稳定推送 | WebSocket 断线自动重连 |
| 零配置部署 | Docker 一键启动 |

## 2. 状态机设计

### 2.1 状态定义

| 状态 | 含义 | 触发条件 | 退出条件 |
|------|------|----------|----------|
| `IDLE` | 空闲，等待请求 | 初始状态 / 任务完成 | 收到 `dispatching` |
| `THINKING` | LLM 思考中 | 检测到 `dispatching` | `Started streaming` |
| `REPLYING` | LLM 回复中 | 检测到 `Started streaming` | `Closed streaming` + `dispatch complete` |
| `EXECUTING` | 工具执行中 | 检测到 `[tools] xxx executing` | 工具完成/失败 |
| `ERROR` | 错误状态 | 检测到 `ERROR` 或 `[tools] xxx failed` | 手动重置 / 超时 |

### 2.2 状态转换图

```
                    ┌──────────────────────────────────────┐
                    │                                      │
                    ▼                                      │
              ┌─────────┐                                  │
              │  IDLE   │◄─────────────────────────────────┤
              └────┬────┘                                  │
                   │ dispatching                           │
                   ▼                                       │
            ┌────────────┐                                 │
            │  THINKING  │                                 │
            └─────┬──────┘                                 │
                  │                                        │
        ┌─────────┼─────────┐                              │
        │         │         │                              │
        ▼         ▼         ▼                              │
  ┌──────────┐ ┌──────────┐ ┌──────────┐                   │
  │ REPLYING │ │EXECUTING │ │  ERROR   │                   │
  └────┬─────┘ └────┬─────┘ └────┬─────┘                   │
       │            │            │                         │
       └────────────┴────────────┴─────────────────────────┘
                    streaming closed / dispatch complete
```

### 2.3 日志事件映射规则

| 日志模式 | 状态转换 | 元数据提取 | 优先级 |
|----------|----------|------------|--------|
| `dispatching` | → THINKING | - | P0 |
| `Started streaming` | THINKING → REPLYING | - | P0 |
| `Closed streaming` + `dispatch complete` | REPLYING → IDLE | - | P0 |
| `[tools] (\w+) executing` | → EXECUTING | tool_name, action | P0 |
| `[tools] (\w+) failed` | → ERROR | tool_name, error_msg | P0 |
| `ERROR` 级别日志 | → ERROR | error_detail | P1 |

## 3. 技术方案

### 3.1 Sidecar 服务（远端）

**技术栈**: Python 3.10+

**核心模块**:

```
sidecar/
├── __init__.py
├── main.py              # 服务入口，启动协调器
├── config.py            # 配置管理，支持环境变量
├── log_reader.py        # 日志流读取器
├── parser.py            # 日志解析器 + 规则引擎
├── state_machine.py     # 状态机实现
├── ws_server.py         # WebSocket 服务
├── health.py            # 健康检查端点
└── rules/
    ├── __init__.py
    ├── base.py          # 规则基类
    └── openclaw_v1.py   # OpenClaw 规则集（版本化）
```

**核心组件选型**:

| 组件 | 技术选型 | 备选方案 | 选择理由 |
|------|----------|----------|----------|
| 日志捕获 | `subprocess` + `tail -F` | `docker logs -f`, `journalctl -f` | 通用性强，支持文件/容器 |
| 日志解析 | 正则表达式 + 规则引擎 | - | 灵活，可版本化 |
| 状态机 | 自定义状态机类 | `transitions` 库 | 轻量，无额外依赖 |
| WebSocket Server | `websockets` | `FastAPI` + `WebSocket` | 轻量级，标准库 |
| 配置管理 | PyYAML + 环境变量 | JSON, TOML | 人类可读，支持注释 |

### 3.2 CLI 客户端（本地）

**技术栈**: Python 3.10+ + Rich

**核心模块**:

```
cli/
├── __init__.py
├── main.py              # CLI 入口 (Typer)
├── ws_client.py         # WebSocket 客户端 + 重连逻辑
├── ui_renderer.py       # 终端 UI 渲染 (Rich)
├── tunnel.py            # SSH 隧道管理
└── config.py            # 配置管理
```

**核心组件选型**:

| 组件 | 技术选型 | 备选方案 | 选择理由 |
|------|----------|----------|----------|
| CLI 框架 | `Typer` | `click`, `argparse` | 现代，支持自动补全 |
| WebSocket Client | `websockets` | - | 与 Server 端一致 |
| 终端 UI | `Rich` | `Textual`, `Blessed` | 简单，美观，学习曲线低 |
| SSH 隧道 | `paramiko` | 系统 `ssh` 命令 | 跨平台，可编程 |

## 4. 接口设计

### 4.1 WebSocket 消息格式

**事件消息 (Server → Client)**:

```json
{
  "type": "state_change",
  "version": "1.0",
  "timestamp": "2024-01-15T10:30:00.123Z",
  "instance_id": "openclaw-gateway-1",
  "data": {
    "state": "EXECUTING",
    "previous_state": "THINKING",
    "meta": {
      "tool_name": "browser",
      "action": "navigate",
      "params": {"url": "https://example.com"},
      "raw_log": "[tools] browser executing: navigate"
    }
  }
}
```

**心跳消息 (Server → Client)**:

```json
{
  "type": "heartbeat",
  "timestamp": "2024-01-15T10:30:05.000Z",
  "instance_id": "openclaw-gateway-1",
  "data": {
    "state": "IDLE",
    "uptime_seconds": 3600,
    "events_processed": 1523
  }
}
```

**错误消息 (Server → Client)**:

```json
{
  "type": "error",
  "timestamp": "2024-01-15T10:30:10.500Z",
  "instance_id": "openclaw-gateway-1",
  "data": {
    "error_code": "TOOL_EXECUTION_FAILED",
    "message": "Command aborted",
    "details": "...",
    "recoverable": true
  }
}
```

**客户端确认 (Client → Server)**:

```json
{
  "type": "ack",
  "message_id": "abc123",
  "received_at": "2024-01-15T10:30:00.150Z"
}
```

### 4.2 HTTP 管理接口

| 端点 | 方法 | 认证 | 说明 | 响应示例 |
|------|------|------|------|----------|
| `/health` | GET | 无 | 健康检查 | `{"status": "healthy", "uptime": 3600}` |
| `/metrics` | GET | 无 | Prometheus 指标 | `sidecar_state_total{state="idle"} 10` |
| `/rules/reload` | POST | JWT | 热加载规则 | `{"status": "ok", "rules_loaded": 5}` |
| `/config` | GET | JWT | 当前配置 | `{"log_source": "...", "ws_port": 8765}` |

### 5.2 Docker 部署（推荐）

**docker-compose.yml**:

```yaml
version: '3.8'

services:
  openclaw-gateway:
    image: openclaw/gateway:latest
    container_name: openclaw-gateway
    restart: unless-stopped
    # OpenClaw 配置...

  sidecar-monitor:
    build:
      context: ./sidecar
      dockerfile: ../deploy/Dockerfile.sidecar
    container_name: sidecar-monitor
    ports:
      - "8765:8765"
    volumes:
      - ./rules:/app/rules:ro
      - /var/log/openclaw:/var/log/openclaw:ro
    environment:
      - OPENCLAW_LOG_SOURCE=stream:/var/log/openclaw/gateway.log
      - WS_PORT=8765
      - WS_HOST=0.0.0.0
      - JWT_SECRET=${JWT_SECRET:-change-me-in-production}
      - LOG_LEVEL=INFO
    depends_on:
      - openclaw-gateway
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "curl", "-f", "http://localhost:8765/health"]
      interval: 30s
      timeout: 10s
      retries: 3
```

**Dockerfile.sidecar**:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

# 安装依赖
COPY requirements-sidecar.txt .
RUN pip install --no-cache-dir -r requirements-sidecar.txt

# 复制代码
COPY sidecar/ ./sidecar/
COPY rules/ ./rules/

# 健康检查
RUN apt-get update && apt-get install -y curl && rm -rf /var/lib/apt/lists/*

EXPOSE 8765

CMD ["python", "-m", "sidecar.main"]
```

## 6. 安全设计

### 6.1 认证机制

| 场景 | 方案 | 实现方式 |
|------|------|----------|
| 开发/内网 | SSH 隧道 | `ssh -L` 端口转发，不暴露公网 |
| 生产环境 | JWT Token | WebSocket 握手时 `Authorization: Bearer <token>` |
| 公网暴露 | TLS + JWT | WSS 加密 + JWT 认证 |

**JWT Token 生成示例**:

```python
import jwt
from datetime import datetime, timedelta

def generate_token(secret: str, instance_id: str) -> str:
    payload = {
        "instance_id": instance_id,
        "exp": datetime.utcnow() + timedelta(hours=24),
        "iat": datetime.utcnow()
    }
    return jwt.encode(payload, secret, algorithm="HS256")
```

### 6.2 推荐配置

**生产环境配置 (config.prod.yaml)**:

```yaml
security:
  auth_enabled: true
  auth_type: jwt
  jwt_secret: ${JWT_SECRET}  # 从环境变量注入
  jwt_expiry_hours: 24

  tls_enabled: true
  tls_cert: /etc/ssl/certs/sidecar.crt
  tls_key: /etc/ssl/private/sidecar.key

  allowed_origins:
    - "ws://localhost:*"
    - "wss://monitor.example.com"

server:
  host: 0.0.0.0
  port: 8765
  max_connections: 10

  ping_timeout: 10
```

### 8.1 风险评估矩阵

| 风险 | 概率 | 影响 | 缓解措施 | 负责人 |

| **多实例隔离** | 中 | 中 | 1. instance_id 区分<br>2. 会话隔离<br>3. 广播/单播模式 | 开发团队 |

### 8.2 日志格式变更应对策略

**规则版本化**:

```
rules/
├── openclaw_v1.0.py    # 2024-01 版本
├── openclaw_v1.1.py    # 2024-03 版本
└── openclaw_v2.0.py    # 2024-06 版本
```

**回退规则**:

```python
class Parser:
    def parse(self, line: str) -> Optional[Event]:
        # 先尝试最新规则
        event = self.rules_v2.match(line)
        if event:
            return event

        # 回退到旧规则
        event = self.rules_v1.match(line)
        if event:
            self.logger.warning(f"Matched old rule: {line}")
            return event

        # 完全匹配失败，发送未知事件
        return UnknownEvent(raw=line)
```
